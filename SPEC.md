# ps-tracker 簡易仕様書

## 1. 概要
- 本アプリは、問題集を任意の階層構造（例: 大問 -> 小問 -> 設問）で管理し、学習状態を追跡する React + Vite アプリ。
- 階層の深さ・構成は固定せず、ユーザーが自由に作成/変更できる。
- データはブラウザ `localStorage` に保存し、サーバー連携は行わない。

## 2. 技術構成
- フロントエンド: React 19
- ビルド: Vite 7
- スタイリング: Tailwind CSS
- PWA: `vite-plugin-pwa`

## 3. データモデル
- 問題集 (`set`)
  - `id`, `name`
  - `rootIds`: 最上位ノードID配列
  - `nodes`: ノード辞書
- ノード (`node`)
  - `id`, `title`
  - `parentId`（最上位は `null`）
  - `childrenIds`（子ノードID配列）
  - `status`: `n`(未), `x`(誤答), `d`(要復習), `c`(完了)
  - `tags`, `notes`
  - `attempts`: 試行履歴（`outcome`, `at`）
  - `firstCheckedAt`, `attemptsToCheck`, `lastSeenAt`

## 4. 主要機能
- 問題集管理
  - 追加、名前変更、削除
- 階層管理
  - 最上位ノード追加
  - 選択ノード配下へ子ノード追加
  - ノード名変更
  - ノード削除（配下を再帰的に一括削除）
- 学習管理
  - ノード単位でステータス更新（`×`, `△`, `✔`）
  - 試行履歴表示
  - メモ表示/編集（状態更新経由で保存）
- 表示機能
  - 階層ツリー表示（クリックでフォーカス切替）
  - フォーカス範囲の状態内訳
  - 「今日やる」候補（優先度ベース）
  - 一覧検索（項目名/パス/メモ/タグ）
  - フィルタ（状態）とソート（優先度/最終学習/誤答数）
  - 空状態表示
- データ入出力
  - JSONエクスポート / JSONインポート
- 操作履歴
  - Undo / Redo

## 5. ステータス遷移
- 初期状態は `n`（未）。
- ノードカードから以下を実行可能:
  - `× にする` -> `x`
  - `△ にする` -> `d`
  - `✔ 完了にする` -> `c`
- 変更時は履歴に追記し、`lastSeenAt` を更新。
- 初めて `c` 到達時のみ `firstCheckedAt` と `attemptsToCheck` を確定。

## 6. 優先度ロジック（概要）
- 基本方針: 未完了（`n/x/d`）を完了（`c`）より優先。
- 考慮要素:
  - 最終学習が古い項目ほど優先
  - 誤答回数が多い項目ほど優先
- 「今日やる」は優先順に `dailyTarget` 件を抽出。

## 7. 保存・互換性
- 保存キー: `ps_tracker_v4`
- 旧キー `ps_tracker_v3` のデータがある場合は読み込み時に移行。
- 旧構造（章/問題）データも読み込み時にツリー構造へ自動変換。

## 8. 画面構成
- ヘッダー
  - 問題集選択、問題集編集、階層編集、Undo/Redo、JSON入出力
- 上段
  - 階層ツリー、状態内訳
- 中段
  - 今日やる
- 下段
  - 検索/フィルタ/ソート付き項目一覧
- フッター
  - ローカル保存に関する注記

## 9. 既知の制約
- バックエンド同期、ユーザー認証、マルチデバイス同期は未対応。
- データは同一ブラウザ環境に閉じる。
