# ps-tracker 簡易仕様書

## 1. 概要
- 本アプリは、問題集ごとに「章」と「問題」の学習状況を管理する React + Vite アプリ。
- データはブラウザ `localStorage` に保存し、サーバー連携は行わない。
- アプリ名は「問題集トラッカー」、PWA として動作可能。

## 2. 技術構成
- フロントエンド: React 19
- ビルド: Vite 7
- スタイリング: Tailwind CSS
- PWA: `vite-plugin-pwa`

## 3. データモデル
- 問題集 (`set`)
  - `id`, `name`, `chapters`, `problems`
- 章 (`chapter`)
  - `id`, `name`
- 問題 (`problem`)
  - `id`, `title`, `chapterId`
  - `status`: `n`(未), `x`(誤答), `d`(要復習), `c`(完了)
  - `tags`, `notes`
  - `attempts`: 試行履歴（`outcome`, `at`）
  - `firstCheckedAt`: 初回完了日時
  - `attemptsToCheck`: 完了到達までの試行回数
  - `lastSeenAt`: 最終操作日時

## 4. 主要機能
- 問題集管理
  - 追加、名前変更、削除
  - 最後の問題集を削除した場合はデモ問題集を再作成
  - JSONエクスポート / JSONインポート
- 章管理
  - 追加（複数章一括追加）、名前変更、削除
  - 章削除時は配下問題も同時削除
- 問題管理
  - 章ごとに問題を複数追加
  - 問題削除
  - ステータス更新（`×`, `△`, `✔︎`）
  - 試行履歴表示
  - メモ表示/編集（状態更新経由で保存）
- 表示機能
  - 章の進捗バー（完了数/総数）
  - 状態内訳（未/×/△/✔︎）
  - 「今日やる」問題リスト（優先度ベース）
  - 問題一覧の検索（問題名/メモ/タグ）
  - 問題一覧のフィルタ（状態）とソート（優先度/最終学習/誤答数）
  - 空状態表示（問題なし / 検索一致なし）
- 操作履歴
  - Undo / Redo

## 5. ステータス遷移
- 初期状態はすべて `n`（未）。
- 問題カードから以下を実行可能:
  - `× にする` -> `x`
  - `△ にする` -> `d`
  - `✔︎ にする` -> `c`
- ステータス変更時は試行履歴に追記し、`lastSeenAt` を更新。
- 初めて `c` になった時のみ `firstCheckedAt` と `attemptsToCheck` を確定。

## 6. 優先度ロジック（概要）
- 基本方針: 未完了（`n/x/d`）を完了（`c`）より優先。
- 考慮要素:
  - 最終学習が古い問題ほど優先
  - 誤答回数が多い問題ほど優先
- 「今日やる」は優先順に `dailyTarget` 件まで抽出。
  - 完了問題の比率を抑えるロジックあり。

## 7. 保存・復元
- `localStorage` キー: `ps_tracker_v3`
- 起動時:
  - キーが存在しない、または JSON パース失敗時はデモ問題集を初期表示。
  - インポート時はJSONを正規化して取り込み、必須項目を補完。
- 更新時:
  - 問題集配列全体を都度 `localStorage` へ保存。

## 8. 画面構成
- ヘッダー
  - 問題集選択、章選択、問題集編集、章編集、Undo/Redo、JSON入出力
- 進捗サマリ
  - 章進捗カード、状態内訳カード
- 今日やる
  - 目標件数入力と候補カード一覧
- 問題一覧
  - 検索/フィルタ/ソートを含む固定操作バーと問題カード一覧
- フッター
  - ローカル保存に関する注記

## 9. 既知の制約（実装準拠）
- バックエンド同期、ユーザー認証、マルチデバイス同期は未対応。
- データは同一ブラウザ環境に閉じる。
- README はテンプレート内容のままで、機能説明は本仕様書が実態に近い。
